# 2026-04-26 工作日志

## 代码审查 - LoveLin 地理相册

完成对整个 Map_Picture 项目的代码审查，项目已从 Node.js 迁移至 Java 21 + Spring Boot 3.3.4 + MySQL。

### 发现的主要问题（按优先级）

**高优先级（需修复）**：
1. `AlbumService.deleteFolder()` - `@Transactional` 内删磁盘文件，事务回滚时文件已丢失，数据库记录还在 → 应收集路径，事务提交后再删
2. `auth.js restoreSession()` - 不检查 `expiresAt`，冷启动时过期 token 仍被注入，首请求 401 失败

**中优先级**：
3. `place/index.js` - `displayPath` 为空时图片空白且点击无效，缩略图下载失败无降级
4. `WebConfig.java` - `corsOrigin` 为空时允许所有 CORS 来源，生产环境需配置
5. `config.js` - prod API_BASE 仍为 `https://example.com` 占位

**低优先级（代码质量）**：
6. `savePhoto()` - 先 INSERT `file_url=''` 再 UPDATE，可考虑删掉 `file_url` 列（前端客户端自行拼 URL）
7. `mapPhotoRow()` - 构造了 `filePath` 字段但 place 页面不直接使用
8. `randomCode(n)` - 奇数 n 时长度实为 n-1（目前仅传偶数，无实际影响）

**综合评分：7.5/10**

### 已修复问题

1. ✅ `deleteFolder()` 事务与磁盘文件删除分离 — 拆为 `deleteFolderInDb`(@Transactional) + `deleteFolder`(事务提交后删文件)
2. ✅ `restoreSession()` 增加 `expiresAt` 过期检测 — 过期 token 自动清除缓存返回 null
3. ✅ 缩略图加载失败降级 — 失败时显示占位 UI(⚠+点击重试)，`retryThumbnail()` 重试后自动预览

### 技术栈确认
- 后端：Java 21 + Spring Boot 3.3.4 + JdbcTemplate + MySQL，端口 3000
- 前端：微信小程序，Bearer token 认证，10s 请求超时
- 照片服务：multipart/base64 双模式上传，需鉴权 URL 访问，前端 downloadFile 到本地 temp 路径显示
